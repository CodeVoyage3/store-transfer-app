门店调拨扫码工具 - 使用说明
================================

联网自动同步商品数据，门店扫码自动匹配商品名！


一、功能概览
----------

  - 调拨基本信息：填写单号、调出/调入门店、入库单号、调拨类型、备注、店长、经手人
  - 商品库导入：从 Excel 导入商品数据（含条码、名称、商品编码）
  - 远程商品库同步：启动时自动从云端拉取加密商品数据，扫码自动匹配商品名
  - 箱管理：多箱分箱管理，切换/新建箱子
  - 扫码录入：摄像头扫码 + 手动输入条码，同单号同箱同条码自动叠加数量
  - 记录管理：分组查看、搜索筛选、编辑删除、双重确认清空
  - 分享/下载：分享为 CSV（钉钉兼容），下载为正式 XLSX（带合并单元格）
  - 自动保存：数据实时存入浏览器 localStorage
  - 自动清空：超过 15 天未打开页面，自动静默清空所有数据
  - 数据加密传输：商品数据 AES-256 加密存储和传输，门店无法直接查看全量数据


二、快速上手
----------

第一步：填写调拨基本信息

  点击"扫一扫"图标弹出调拨信息填写弹窗，填写完成后自动进入扫码页。

  必填项：调拨单号、调出门店、调入门店、调拨类型、店长、经手人

  示例：
    调拨单号：RVKKT20240601-001
    调出门店：东莞壹号广场KKV店
    调入门店：东莞厚街万达广场KKV店
    门店入库单号：TPA4000001298231

  锁定态下只显示单号和调出/调入信息，点击"修改"可查看和编辑全部字段。


第二步：导入商品库

  在扫码页的"商品库"区域点击"导入"，选择 Excel 文件。

  商品库为空时页面会提示"暂无商品，请先导入 Excel 商品信息"。

  Excel 格式要求：
    - 必须包含"条形码"（或"条码"）列和"商品名称"（或"名称"）列
    - 可选列："商品编码"、"基本单位"（或"单位"）
    - 支持多个条码用逗号分隔（自动拆分为多条）
    - 自动跳过重复条码和以 ZB 结尾的条码

  导入成功后自动提示导入数量。也可点击"清空"双重确认后清空商品库。


第三步：扫码录入

  1. 点击"扫一扫"图标启动摄像头
  2. 将条码对准扫描框，扫码成功后有震动+提示音
  3. 输入数量后自动提交（按回车或失焦）

  手动输入条码：在下方的输入框输入条码，点击"查找"按钮。

  叠加逻辑：同一调拨单号 + 同一箱号 + 同一条码 -> 自动累加数量
  例：A商品3件录入第1箱，再扫A商品1件入第1箱 -> 自动合计为4件。


第四步：箱管理

  - 新建箱：点击新建箱子，自动切换到新箱
  - 左右箭头切换：切换已创建的箱子
  - 状态显示：显示当前箱的记录条数和总件数

  必须先在第1箱有记录后，才能新建第2箱。


第五步：查看记录

  切换到"记录"Tab，可查看所有调拨记录。

  记录分组：按调拨单号 -> 按箱号 层级折叠展示。

  功能：
    - 搜索：按商品名称/条码/门店搜索
    - 筛选：按调入门店筛选
    - 点击记录可编辑（修改条码/数量/箱号）
    - 删除单条记录或整张单据（双重确认）
    - 勾选单据后批量导出


第六步：导出数据

  在记录页勾选要导出的单据后：

    分享按钮 -> CSV 格式，可分享到微信/钉钉等，直接查看
    下载按钮 -> XLSX 格式，正式 Excel 文件，带合并单元格

  导出列说明：
    门店调拨单号 | 货源门店 | 目标门店 | 创建时间 | 调拨类型 | 门店入库单号
    | 备注 | 店长 | 经手人 | 商品编码 | 商品条形码 | 商品名称 | 实调数量
    | 总计 | 仓库 | 箱号

  同单据的非明细字段（单号、门店、时间、类型、入库单号、备注、店长、
  经手人、总计）自动合并居中。


三、远程商品库同步（新功能）
----------------------

门店 App 启动时会自动从云端拉取加密的商品数据，扫码时自动匹配商品名称。

  - 数据由管理员统一维护和发布，门店无需接触商品原始Excel
  - 数据 AES-256 加密传输和存储，门店无法直接查看全量商品数据
  - 扫码匹配速度快（本地缓存，<1ms），支持离线使用
  - 显示"📦 N 条 已同步"表示同步成功


四、管理员工具
----------

管理工具文件：管理工具.html（在电脑浏览器打开）

管理员操作流程：
  1. 打开 管理工具.html，输入管理密码进入
  2. 配置 GitHub Token（保存后下次自动填充）
  3. 上传商品 Excel → 自动识别条码列和商品名列
  4. 确认数据预览（只保留条码和商品名，已脱敏）
  5. 点击"加密并发布"，数据AES加密后自动推送到云端
  6. 门店下次打开 App 自动获取最新数据

  - 发布记录会保留在浏览器中，方便追踪每次更新
  - 仅管理员可在电脑上操作，普通门店没有此页面


五、数据安全
----------

存储方式：
  全部数据存在手机浏览器本地（localStorage），不上传任何服务器，
  不联网也能正常使用。

自动清空规则：
  页面记录上次访问时间，超过15天未打开页面，自动静默清空商品库和
  调拨记录。这是为了保障数据时效性，避免过期数据混淆。

手动备份/恢复：
  右上角"自动保存"指示正常（绿色）/异常（黄色），异常时需手动
  导出备份。

注意事项：
  - 扫码需授予摄像头权限
  - 支持条码：Code 128 / EAN-13 / EAN-8 / UPC / Code 39 / QR 码等


六、安装到桌面（推荐）
------------------

Android（Chrome）：打开页面 -> 菜单 ⋮ -> 添加到主屏幕
iOS（Safari）：打开页面 -> 分享 -> 添加到主屏幕

添加到桌面后可像原生应用一样全屏运行，体验更好。


七、技术说明
----------

  - 纯 HTML + CSS + JavaScript 单文件应用
  - 扫码引擎：浏览器 BarcodeDetector API（首选）+ QuaggaJS（备选）
  - XLSX 导出：SheetJS（xlsx.core.min.js）
  - 无任何后端服务依赖
  - PWA 支持（manifest.json + 图标）
